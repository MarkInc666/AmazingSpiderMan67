Spider-Man Pinball - Mini-Wizard Mode Theme Ideas

Core Theme
-----------

Each mini-wizard should not just mean “after X villains.” It should feel like Spider-Man dealing with the larger consequences of all the villain chaos.

Good overall structure:

Villains cause trouble
→ Spider-Man fights enough of them
→ Their combined chaos triggers a bigger city-wide crisis
→ Mini-wizard mode resolves that crisis
→ Next chapter unlocks tougher villains

This works whether the game ends up with 10 villains, 15 villains, or 25+ villains.


Mini-Wizard Theme Ideas
-----------------------

1. Daily Bugle Special Report

Theme:
J. Jonah Jameson turns the villain chaos into a city-wide panic headline and blames Spider-Man.

Player-facing callout:
DAILY BUGLE SPECIAL REPORT!
SPIDER-MAN BLAMED FOR CITY CHAOS!
CLEAR SPIDER-MAN'S NAME!

Mode idea:
A multiball where the player has to clear Spider-Man's name by collecting evidence from major shots.

Possible shot mapping:
- Web targets = collect photos/evidence
- Spinner = print presses / headline value
- Saucers = deliver story to the Bugle
- Rooftop = chase the real culprit
- Daily Bugle VUK = publish the truth / major jackpot

Why it works:
This fits the Daily Bugle VUK and mystery award idea. It also gives the first mini-wizard a funny, very 1967 cartoon feeling instead of making it just another villain fight.


2. Sinister Syndicate Attack

Theme:
The defeated villains regroup and launch a coordinated attack.

Player-facing callout:
THE SINISTER SYNDICATE STRIKES!

Mode idea:
A 2-ball or 3-ball multiball where previously played villains each relight one major shot. Each jackpot represents one defeated villain trying to get revenge.

Possible shot mapping:
- Rhino = drop targets
- Vulture = upper spinner
- Lizard = web targets
- Sandman = right bank
- Electro = flashing roving shot
- Mysterio = mystery / fake-out awards
- Doc Ock = rollovers or controlled arms

Why it works:
This is the cleanest “you played several villains, now they all come back at once” concept. It also lets the game reuse the villains the player actually chose, which makes the mode feel personalized.


3. Rooftop Rescue

Theme:
The city is in danger and Spider-Man has to swing across rooftops rescuing civilians.

Player-facing callout:
ROOFTOP RESCUE!
SAVE THE CITY!

Mode idea:
A fast multiball focused on the upper playfield.

Possible shot mapping:
- Upper spinner = swing across the skyline
- Upper targets = rescue civilians
- Left exit = return to danger zone
- Right exit = save civilians / collect jackpot
- Daily Bugle VUK = emergency rescue award

Why it works:
The upper playfield already represents sky/rooftops, so this mode makes the custom part of the machine feel important. This would be a great second mini-wizard once the player understands upper playfield flow.


4. Web of Crime

Theme:
Spider-Man discovers that the villains are connected through a larger criminal network.

Player-facing callout:
THE WEB OF CRIME REVEALED!

Mode idea:
Each major shot lights as a clue. Collect enough clues to reveal the mastermind behind the chapter.

Possible shot mapping:
- Inlanes = trace the clue
- Drops = break up the gang
- Spinner = follow the trail
- Saucers = interrogate informants
- Web targets = web up the criminals
- VUK = connect the evidence

Why it works:
This is a good bridge toward Kingpin. It makes the full game feel like it is building toward organized crime instead of being a random villain-of-the-week collection.


5. Kingpin's Grip

Theme:
Kingpin is not the final wizard yet, but his influence starts showing up between chapters.

Player-facing callout:
KINGPIN'S GRIP TIGHTENS!

Mode idea:
A mini-wizard where Kingpin's men interfere while Spider-Man fights through the city.

Possible shot mapping:
- Drop banks = smash the gangs
- Saucers = raid hideouts
- Spinner = chase getaway cars
- Web targets = collect evidence
- Rooftop = escape ambush / super jackpot

Why it works:
This lets Kingpin slowly become the final threat. You can tease him over multiple mini-wizards before the final wizard.

-----------------------

Mini-Wizard 1: Daily Bugle Special Report

After the first batch of villains, J. Jonah Jameson blames Spider-Man.

Callout:
DAILY BUGLE SPECIAL REPORT!

This should be lighter, funny, and very 1967 cartoon.

Callout:
ROOFTOP RESCUE!

This should lean into the upper playfield, spinner, exits, and VUK.


Mini-Wizard 3: Sinister Syndicate

After more villains are played, several of them team up.

Callout:
THE SINISTER SYNDICATE STRIKES!

This should feel like a bigger, more dangerous multiball using shots tied to previously played villains.


Mini-Wizard 4: Kingpin's Grip

The pattern behind the villain chaos is revealed.

Callout:
KINGPIN'S GRIP TIGHTENS!

This should point directly toward the final wizard.


Final Wizard: Kingpin

Callout:
KINGPIN'S EMPIRE FALLS!

This is the final showdown and should use the whole playfield.


Clean Story Escalation

Public panic
→ Rooftop rescue

→ Organized crime reveal

Strong Design Recommendation
----------------------------

The individual modes are already villain fights. The mini-wizards should be bigger city-level events:

- Bugle scandal
- Rooftop rescue
- Sinister team-up
- Crime wave
- Kingpin takeover

That gives the game better variety and makes each chapter feel like the story is escalating.

Player-facing rule idea:

Survive each crisis to reveal stronger villains.
Clear all chapters to face Kingpin.

# Villain Start / Saucer System

The 3-bank drop targets control the three saucers.

Drop targets physically reset at the start of each ball and after each villain mode.

Saucer progress is stored per player and carries between balls, but resets after each villain mode. This means that after playing a villain, the player must hit drops again to qualify the next villain start.

As the chapter progresses, fewer villains remain unplayed. By the 5th villain in a chapter, only one valid villain remains, so lighting any saucer to state 1 is enough to start the final villain of that chapter.

Each saucer has a state from 0 to 5:

```text
0 - Unlit. Saucer awards points only.
1 - Slow flash. Saucer starts the next available villain immediately.
2 - Slow/medium flash. Saucer starts villain select with up to 2 choices.
3 - Medium flash. Saucer starts villain select with up to 3 choices.
4 - Medium/fast flash. Saucer starts villain select with up to 4 choices.
5 - Fast flash. Saucer starts villain select with up to 5 choices.
```

During each drop-bank cycle, hitting a drop target advances the corresponding saucer state by 1, up to a maximum of 5.

When all 3 drops have been hit, the bank is complete and resets. The player may then build the saucer states again.

After a saucer reaches state 5, further hits do not increase that saucer state. They only score points, with value increasing based on bank completions.

If a saucer has only one valid unplayed villain available, it starts that villain immediately.

If a saucer has two or more valid unplayed villains available, it starts the villain select carousel.

Villains are grouped into chapters of 5. After all 5 villains in the chapter have been played, the chapter mini-wizard becomes available.

Once all three saucers are at state 1 or higher, the star rollover advances all three saucers by one state.

---

# Mini-Wizard Start

After 5 villains have been played, the chapter mini-wizard becomes ready.

Shoot the Daily Bugle VUK to start the mini-wizard sequence.

The ball pauses in the VUK while the audio/video intro plays.

The mini-wizard multiball starts.

The ball is kicked to the upper playfield. During the opening phase, the player can take pictures by spinning the upper spinner. After collecting enough pictures, the left exit lights for add-a-ball.

Proposed rule:

```text
Left exit = add-a-ball
```

## One important design note

Your star rollover rule is good, but I would add one guardrail:

```text
The star rollover only advances saucers that are already at state 1 or higher.
```

That matches your wording and keeps it from being too powerful. So if the saucers are:

```text
Left: 1
Middle: 1
Right: 0
```

the star does **nothing** or maybe scores points only.

If the saucers are:

```text
Left: 1
Middle: 1
Right: 1
```

then star advances them to:

```text
Left: 2
Middle: 2
Right: 2
```

That makes the star a reward for building all three saucers first, not a shortcut from zero.

# Case File System

The right 5-bank represents **Daily Bugle case files** or **Spider-Man preparing for the next villain**.

Each target corresponds to an upgrade type:

```text
Target 1 - Extra Jackpot
Target 2 - More Time
Target 3 - Higher Multiplier
Target 4 - Safer Mode / Ball Save / Hold
Target 5 - Super Jackpot Boost
```

The lower spinner cycles which target is lit.

When the player hits the lit target, they collect that upgrade for the next villain mode.

---

## Why this works

It creates a nice player choice:

```text
Do I start the villain now?
Or do I shoot spinner/5-bank first to make the villain mode stronger?
```

That is good pinball design because it creates risk/reward without blocking progression.

You already have:

```text
3-bank drops = qualify/start villains
A+B = Daily Bugle / gate / photos
```

So this gives:

```text
lower spinner + 5-bank = improve villain modes
```

That is a clean third lane of strategy.

# How upgrades could affect the 5 chapter villains

For a chapter of 5 villains, each target could map to a specific villain **or** to a general upgrade.

I prefer **general upgrades**, because it scales better when you add many villains.

## General upgrade examples

### Upgrade 1: Extra Jackpot

```text
Adds one additional jackpot opportunity to the next villain mode.
```

Examples:

```text
Rhino: one extra Rhino Bash jackpot
Lizard: one extra serum delivery
Sandman: one extra flashing target award
Doc Ock: one extra jackpot before timer pressure increases
```

### Upgrade 2: More Time

```text
Adds extra time to the next timed villain mode.
```

Examples:

```text
Vulture: +10 seconds
Lizard: +5 seconds before serum value starts falling
Electro: +2 seconds per spark
Scorpion: +2 seconds to hit the final drop
```

### Upgrade 3: Multiplier Boost

```text
Starts the next villain mode with a scoring multiplier or higher base value.
```

Examples:

```text
Rhino: rage scoring starts one level higher
Vulture: spinner starts at 2x
Sandman: flashing target starts at 75K instead of 50K
Electro: first spark jackpot doubled
```

### Upgrade 4: Safety

```text
Makes the next villain mode easier or safer.
```

Examples:

```text
Short ball save at villain start
One free miss on timed sequence
Doc Ock timer delayed
Scorpion gets one retry
```

### Upgrade 5: Super Jackpot Boost

```text
Adds or increases the mode’s big finish award.
```

Examples:

```text
Rhino: bigger final bash jackpot
Vulture: bonus for max target colors
Lizard: bigger final serum award
Sandman: bigger order-completion jackpot
```

# Better: let the player stack up to 3 Case Files

I would not let every upgrade stack forever. That gets hard to balance.

Use a cap:

```text
Player may carry up to 3 Case Files into the next villain mode.
```

Then when a villain starts:

```text
Apply collected Case Files.
Clear Case Files.
```

That creates good pre-villain strategy without letting the player over-build one mode.

```text
Collected:
```

# Spinner cycling behavior

The lower spinner can cycle the lit 5-bank target one step per spin:

```text
Spin 1 → Target 1 lit
Spin 2 → Target 2 lit
Spin 3 → Target 3 lit
Spin 4 → Target 4 lit
Spin 5 → Target 5 lit
Spin 6 → Target 1 lit again
```

Or, if that cycles too fast, use every 3 spins:

```text
Every 3 lower-spinner spins advances the lit Case File target.
```

I’d start with **every 3 spins**. One step per spin may be too twitchy.

---

# Target behavior

When the lit 5-bank target is hit:

```text
Collect that Case File upgrade
Turn off that target’s upgrade light
Light/cycle to the next available Case File target
```

You have two choices:

## Option A: Collected upgrade target stays unlit until villain starts

This means each upgrade can only be collected once before the next villain.

```text
Clean and controlled.
```

## Option B: Target can relight after the bank resets

This allows stacking duplicates.

```text
More exciting, but harder to balance.
```

I’d use **Option A**.

# How to make it scalable in code

Do not hard-code the five upgrades differently in the qualify system.

The qualify system should only know:

```text
Player collected:
- extra_jackpot
- more_time
- multiplier
- safety
- super_boost
```

Then each villain mode applies them.

Example in a villain mode:

```python
def mode_start(self, **kwargs):
    super().mode_start(**kwargs)

    self.extra_jackpot = int(kwargs.get("extra_jackpot", 0))
    self.more_time = int(kwargs.get("more_time", 0))
    self.multiplier_boost = int(kwargs.get("multiplier", 0))
    self.safety = int(kwargs.get("safety", 0))
    self.super_boost = int(kwargs.get("super_boost", 0))

    self._apply_case_files()
```

Then:

```python
def _apply_case_files(self):
    self.max_jackpots = self.base_max_jackpots + self.extra_jackpot
    self.mode_timer_seconds = self.base_timer_seconds + (10 * self.more_time)
    self.jackpot_multiplier = 1 + self.multiplier_boost
    self.safety_available = self.safety > 0
    self.super_jackpot_value = self.base_super_jackpot + (250000 * self.super_boost)
```

Some villains will override the exact meaning, but the shape stays the same.
