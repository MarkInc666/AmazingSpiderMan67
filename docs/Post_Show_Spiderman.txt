You’d basically have **three persistent zones**:

```text
LEFT PANEL
Villain / Chapter Progress

CENTER PANEL
Current action / active mode / priority messages

RIGHT PANEL
Daily Bugle / Case Files / Wizard Prep
```

The important rule should be:

> Left and right panels are persistent status. Center panel is interrupt-driven gameplay messaging.

That means the player can always glance left/right for progress, while the center gets used for whatever is urgent.

## Center: active message layer

The center should be where the game “talks” to the player:

```text
RHINO BASH
RAGE LEVEL 3

```

or:

```text
BALL SAVED
```

or:

```text

```

or:

```text

```

I would treat this as a **priority stack**, because some messages matter more than others.

Suggested priority order:

```text
1. Tilt / Ball Saved / Extra Ball / Match-level urgent alerts
2. Skill Shot / Mode Start / Mode Complete / Bookend
3. Active villain mode widget
4. Timed callouts: Chaos Level 3, Jackpot Lit, Hit B to collect Bugle
5. Idle/default center display
```

So a “Ball Saved” message can temporarily cover the Rhino mode widget, then expire and return to Rhino.

## Best architecture

In Godot, I’d make:

```text
MainDisplay.tscn
  LeftStatusPanel
    VillainChapterWidget

  CenterStage
    ActiveModeContainer
    PopupMessageLayer

  RightStatusPanel
    DailyBugleWidget
    CaseFilesWidget
```

The center should support temporary widgets with timed expiration:

```text
show_skillshot_widget
show_ball_saved_popup
show_extra_ball_popup
show_chaos_level_popup
show_bugle_prompt
show_villain_mode_widget
clear_center_popup
```

In MPF/GMC terms, the left and right widgets can stay loaded almost all the time. The center widgets should be event-driven and often use `expire`.

Example concept:

```yaml
widget_player:
  ball_save_default_saving_ball:
    ball_saved_popup:
      action: play
      expire: 2s

  chaos_level_3_reached:
    chaos_level_3_popup:
      action: play
      expire: 2s

  bugle_b_ready:
    bugle_b_prompt:
      action: play
      expire: 4s
```

For the active villain mode widget, don’t expire it immediately. It should stay until the mode ends or the summary/bookend takes over.

This is a strong layout. It gives you persistent strategy on the sides and immediate gameplay instruction in the middle, which is exactly what a complicated rule set needs.

# Spider-Man Rule System Summary

## Main Game Flow

The game has several connected systems:

```text
A+B / Daily Bugle → opens gate → upper playfield/photos/rewards

3-bank drops → build saucer states → start/select villains

Lower spinner + right 5-bank → collect Case Files → enhance next villain

Pops → build city chaos / chapter value

Star rollover → upgrades saucers once all three are lit

5 villains played → mini-wizard ready

Complete chapters → final Kingpin wizard
```

---

# A + B Rollovers / Daily Bugle System

## Purpose

The A+B rollovers are the **Daily Bugle / upper access system**.

```text
A = DAILY
B = BUGLE
```

Completing A+B opens the gate to the upper playfield.

## Flow

```text
Complete A+B
→ open upper gate
→ reach upper playfield
→ spin upper spinner to take photos
→ shoot Daily Bugle VUK to collect reward / mystery / start mini-wizard sequence
```

## State

This should persist between balls:

```text
A completed
B completed
Daily Bugle progress
Mystery/reward readiness
photos collected
```

The physical gate should probably **not** stay open across balls, but the fact that the gate was qualified can persist.

```text
gate_qualified = persists
gate_physically_open = resets safely
```

---

# 3-Bank Drops / Saucer Villain Start System

## Purpose

The 3-bank drop targets control the three saucers.

```text
left drop → left saucer state
middle drop → middle saucer state
right drop → right saucer state
```

Drop targets physically reset:

```text
at start of each ball
after each villain mode
after completing all 3 drops
```

Saucer progress:

```text
carries between balls
resets after each villain mode
```

## Saucer States

Each saucer has a state from 0 to 5:

```text
0 - Unlit. Saucer awards points only.
1 - Slow flash. Starts next available villain immediately.
2 - Slow/medium flash. Starts villain select with up to 2 choices.
3 - Medium flash. Starts villain select with up to 3 choices.
4 - Medium/fast flash. Starts villain select with up to 4 choices.
5 - Fast flash. Starts villain select with up to 5 choices.
```

## Bank Cycle

During each drop-bank cycle:

```text
hit a drop target
→ advance the corresponding saucer by 1 state
```

When all 3 drops are hit:

```text
bank complete
→ score bank completion
→ reset drops
→ allow saucer states to be built again
```

After a saucer reaches state 5:

```text
further hits do not increase the state
further hits only score points
points increase based on bank completions
```

## Villain Start Logic

If a saucer has one valid unplayed villain available:

```text
start that villain immediately
```

If it has two or more valid villains available:

```text
start villain select carousel
```

If the carousel would only have one playable choice:

```text
skip carousel
start that villain directly
```

---

# Villain Chapters

Villains are grouped into chapters of 5.

```text
Play 5 villains
→ chapter mini-wizard becomes available
```

Recommended rule:

```text
Starting/playing a villain counts toward chapter progress.
Defeating the villain can give better scoring, bonuses, or final wizard value.
```

This keeps progression moving even if the player fails a mode.

---

# Star Rollover

## Purpose

The star rollover is a saucer upgrade shot.

## Rule

Once all three saucers are at state 1 or higher:

```text
star rollover advances all 3 saucers by 1 state
```

Example:

```text
Left saucer: 1
Middle saucer: 1
Right saucer: 1

Star hit →

Left saucer: 2
Middle saucer: 2
Right saucer: 2
```

The star should only work if all three saucers are already lit.

If any saucer is still at 0:

```text
star scores points only
or gives a minor award
```

This makes the star a reward for building the full saucer set, not a shortcut from nothing.

# Right 5-Bank Drops / Case Files

## Purpose

The right 5-bank is the **villain preparation system**.

It lets the player improve the next villain mode before starting it.

## Case File Types

Each target represents one Case File:

```text
1. Extra Jackpot
2. More Time
3. Multiplier Boost
4. Safety / Retry
5. Super Jackpot Boost
```

## Collection Rule

```text
Hit the lit Case File target
→ collect that Case File for the next villain
```

Each Case File type can be collected once before the next villain.

When the next villain starts:

```text
collected Case Files apply to that villain
individual Case Files clear
```

## Every villain supports all five Case File types

Universal meanings:

```text
Extra Jackpot = more things to collect
More Time = less time pressure
Multiplier Boost = better scoring
Safety / Retry = one mistake forgiven
Super Jackpot Boost = better final award
```

Each villain interprets them thematically.

Example:

```text
Rhino:
Extra Jackpot = +1 Rhino Bash jackpot
More Time = longer jackpot timer
Multiplier = rage scoring starts higher
Safety = first miss forgiven
Super Boost = bigger final bash award

Vulture:

More Time = longer Sky Attack timer
Multiplier = spinner starts stronger

Super Boost = bigger max-color award
```

# Complete Case File / Wizard Prep

Collecting all 5 Case Files before a villain gives an extra bonus:

```text
Complete Case File
→ Wizard Prep +1
```

Important:

```text
The five individual Case Files help the next villain.
The all-5 completion bonus does not further help that villain.
The all-5 completion bonus is saved for the mini-wizard.
```

## Wizard Prep

Wizard Prep is stored for the current chapter.

```text
wizard_prep_this_chapter += 1
```

At the mini-wizard, Wizard Prep can improve things like:

```text
mini-wizard jackpot value
ball save time
photos needed for add-a-ball
add-a-ball timer
super jackpot readiness
```

Example mini-wizard prep table:

```text
0 Prep - normal mini-wizard
1 Prep - bigger starting jackpot
2 Prep - longer ball save
3 Prep - fewer photos needed for add-a-ball
4 Prep - add-a-ball shot stays lit longer
5 Prep - super jackpot or add-a-ball starts easier
```

Villain Start / Saucer System

The 3-bank drop targets control the three saucers.

Drop targets physically reset at the start of each ball and after each villain mode.

Saucer progress is stored per player and carries between balls, but resets after each villain mode. 
This means that after playing a villain, the player must hit drops again to qualify the next villain start.

(As the chapter progresses, fewer villains remain unplayed. 
By the 5th villain in a chapter, only one valid villain remains, 
so lighting any saucer to state 1 is enough to start the final villain of that chapter.)

Each saucer has a state from 0 to 5:

0 - Unlit. Saucer awards points only.
1 - Slow flash. Saucer starts the next available villain immediately.
2 - Slow/medium flash. Saucer starts villain select with up to 2 choices.
3 - Medium flash. Saucer starts villain select with up to 3 choices.
4 - Medium/fast flash. Saucer starts villain select with up to 4 choices.
5 - Fast flash. Saucer starts villain select with up to 5 choices.

During each drop-bank cycle, hitting a drop target advances the corresponding saucer state by 1, up to a maximum of 5.

When all 3 drops have been hit, the bank is complete and resets. The player may then build the saucer states again.

After a saucer reaches state 5, further hits do not increase that saucer state. 
They only score points, with value increasing based on bank completions.

If a saucer has only one valid unplayed villain available, it starts that villain immediately.

If a saucer has two or more valid unplayed villains available, it starts the villain select carousel.

Villains are grouped into chapters of 5. After all 5 villains in the chapter have been played, the chapter mini-wizard becomes available.

Once all three saucers are at state 1 or higher, the star rollover advances all three saucers by one state.
